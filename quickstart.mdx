---
title: "Make your first Microsoft Graph API call in minutes"
description: "Run your first Microsoft Graph API query in Graph Explorer in under two minutes — no installation, account, or configuration required to begin."
---

Graph Explorer requires no installation, no account, and no configuration. You can open it right now in your browser and start running Microsoft Graph API queries against sample data within seconds. This guide walks you through opening Graph Explorer, running your first query anonymously, signing in with your Microsoft account, and making an authenticated call to retrieve your own profile data.

<Steps>
  <Step title="Open Graph Explorer">
    Navigate to [https://developer.microsoft.com/en-us/graph/graph-explorer](https://developer.microsoft.com/en-us/graph/graph-explorer) in your browser.

    The interface loads with three main panels: the query bar at the top, the sample queries panel on the left, and the response viewer on the right. No sign-in is required at this point.
  </Step>
  <Step title="Run a query with the demo account">
    The query bar is pre-filled with:

    ```text
    GET https://graph.microsoft.com/v1.0/me
    ```

    Click **Run query**. Graph Explorer sends the request using Microsoft's demo account and returns a sample profile response in the response viewer. The response body, HTTP status code, and response headers all appear on the right side of the screen.

    <Note>
      In demo mode, `/me` returns data for a fictional Microsoft user, not your own account. All responses in this mode reflect pre-populated sample data.
    </Note>
  </Step>
  <Step title="Sign in with your Microsoft account">
    To query your real data, click **Sign in to Graph Explorer** in the top-right corner of the page.

    A Microsoft authentication dialog appears. Enter the credentials for your Microsoft account — personal (outlook.com, hotmail.com), work, or school. After authenticating, your display name and profile photo appear in the header, confirming that Graph Explorer is now operating against your actual tenant.

    <Tip>
      If you do not want to use your primary work account, you can create a free [Microsoft 365 Developer Program](https://developer.microsoft.com/en-us/microsoft-365/dev-program) tenant specifically for testing.
    </Tip>
  </Step>
  <Step title="Run an authenticated query">
    With the query bar still showing `GET https://graph.microsoft.com/v1.0/me`, click **Run query** again.

    This time Graph Explorer calls the real Graph API with your access token. The response body returns your actual Microsoft 365 profile data — including your display name, email address, job title, and user ID.

    If Graph Explorer prompts you to consent to the `User.Read` permission, click **Accept**. This grants Graph Explorer permission to read your profile on your behalf.
  </Step>
  <Step title="Explore sample queries">
    Click the **Sample queries** tab in the left panel. The panel shows a categorized list of more than 100 pre-built queries covering users, mail, calendar, files, Teams, and more.

    Select any sample — for example, **Get my calendar events** under the Calendar category. Graph Explorer populates the query bar with the appropriate endpoint and HTTP method. Click **Run query** to execute it.

    Each sample query also shows the permissions it requires. If you have not yet consented to those permissions, Graph Explorer prompts you to do so before the request succeeds.
  </Step>
</Steps>
