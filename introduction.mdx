Graph Explorer is a browser-based developer tool hosted at [developer.microsoft.com](https://developer.microsoft.com/en-us/graph/graph-explorer) that lets you build and execute HTTP requests against the Microsoft Graph API, inspect responses, manage permissions, and learn the API surface — all directly in your browser. You do not need to install anything, configure a development environment, or register an application to get started. Whether you are evaluating Graph for the first time or debugging a production integration, Graph Explorer gives you immediate, interactive access to the same endpoints your code will call.

## How Graph Explorer fits into the Microsoft ecosystem

Microsoft Graph is the unified REST API gateway for Microsoft 365, Azure Active Directory, and a growing set of Microsoft services including Teams, SharePoint, OneDrive, Outlook, Planner, and Intune. Rather than calling individual product APIs, developers use the single endpoint `https://graph.microsoft.com` to access data and functionality across all of those workloads.

Graph Explorer sits at the entry point of that ecosystem. It is the fastest way to understand what an endpoint returns, what permissions it requires, and how the request must be structured before you write any code. Microsoft's official documentation at [learn.microsoft.com](https://learn.microsoft.com/en-us/graph/) links directly to Graph Explorer for many of its code samples, making it the standard tool for hands-on learning alongside the reference docs.

Graph Explorer also integrates with Microsoft's [Postman collection for Microsoft Graph](/advanced/postman-collection), letting you export queries you build in the browser directly into Postman for use in automated workflows.

## Demo account vs. signed-in mode

Graph Explorer supports two modes of use:

- **Demo account (anonymous):** When you open Graph Explorer without signing in, it connects to a read-only Microsoft-managed demo tenant. You can explore endpoints and inspect realistic sample responses without any credentials. This mode is intentionally limited — write operations are blocked and responses reflect demo data, not your organization's data.

- **Signed-in mode:** When you sign in with a Microsoft account (personal, work, or school), Graph Explorer connects to your real Microsoft 365 tenant. You can read and write your actual data, consent to Graph API permissions interactively, and test queries that will behave exactly as they would in your application.

<Note>
  Signing in with a work or school account that belongs to an organization may require an administrator to grant consent for certain permissions. If a permission requires admin consent, Graph Explorer will indicate this and provide guidance on how to proceed.
</Note>
